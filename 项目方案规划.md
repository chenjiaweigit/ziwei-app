# 紫微命盘小程序 · 项目方案规划

---

## 一、项目定位

基于倪海夏《天纪》体系的紫微斗数命理小程序，以**排盘 + AI 解读**为核心功能，以**合盘分析**为社交裂变点，以**会员体系**为商业化路径。

### 目标用户
- 对紫微斗数感兴趣的入门者（占 70%）
- 已有基础的命理爱好者（占 20%）
- 专业命理师作为工具使用（占 10%）

### 差异化优势
- **倪海夏正宗体系**：严格遵循《天纪》教学体系，不采用飞星派等非正统算法
- **AI 深度解读**：基于 LLM 的自然语言解读，比传统"查表式"解读更个性化
- **开源排盘引擎**：算法公开透明，可验证、可扩展

## 二、技术选型

| 模块 | 选型 | 理由 |
|------|------|------|
| 框架 | Next.js 15 (App Router) | SSR/SSG 支持 SEO，App Router 简洁 |
| 语言 | TypeScript | 类型安全，适合多人协作 |
| 样式 | Tailwind CSS | 快速原型，设计系统统一 |
| 排盘 | iztro + lunar-javascript | 成熟的紫微斗数排盘库 |
| AI | OpenAI 兼容协议（DeepSeek） | DeepSeek 中文能力强，成本低 |
| 状态管理 | React useState + Context | 轻量，无需 Redux |
| 存储 | JSON 文件数据库 | 零配置起步，后期可迁移至 SQLite/PostgreSQL |
| 认证 | JWT | 无状态认证，适合前后端分离 |
| 部署 | Vercel / 自建服务器 | 按需选择 |

## 三、系统架构

```
用户（浏览器）
    │
    ├── Next.js 静态页面（SEO 内容）
    │   ├── 首页 /
    │   ├── 百科 /knowledge/...
    │   └── 古籍 /library/...
    │
    └── Next.js 动态页面 + API
        ├── 排盘 /chart
        │   └── POST /api/generate
        ├── AI 解读 /chart（右侧面板）
        │   └── POST /api/interpret（SSE 流式）
        ├── 合盘 /heming
        │   ├── POST /api/generate（x2）
        │   └── POST /api/heming（SSE 流式）
        └── 用户系统
            ├── POST /api/auth/register
            ├── POST /api/auth/login
            ├── GET /api/auth/me
            ├── GET/POST/DELETE /api/user/history
            └── GET/POST /api/payment
```

### 数据流

```
排盘流程：
  用户填表 → POST /api/generate → generateChart()
    → iztro 排盘 → 空宫借星 → 大限计算 → 格局检测
    → 返回 ZiweiChart JSON → 前端渲染棋盘

AI 解读流程：
  用户点击"解读" → POST /api/interpret
    → 提取命盘上下文 → 组装 prompt → 调 LLM → SSE 流式返回
    → 前端逐步渲染 Markdown

合盘流程：
  双方填表 → 并行 POST /api/generate（x2）
    → POST /api/heming（chartA, chartB）
    → 双盘上下文合并 → 合盘 prompt → LLM 流式返回
```

## 四、核心模块详情

### 4.1 排盘算法（lib/ziwei/）

基于 iztro 库实现，核心流程：

```
输入：公历年月日 + 时辰索引 + 性别
  → 调用 iztro.astro.bySolar() 获取原始盘
  → 映射为本地 Palace[] 结构
  → 空宫借对宫（branch + 6）
  → 计算当前年龄 & 大限
  → 格局检测（detectPatterns）
  → 返回 ZiweiChart
```

### 4.2 AI 解读系统（lib/ai/）

三层架构：
1. **知识层**（knowledge.ts）：从命盘提取结构化上下文（主星/四化/格局/大限）
2. **提示词层**（prompts.ts）：倪海夏体系系统提示词 + 6 大主题模板
3. **推理层**（interpret.ts）：LLM 客户端封装 + SSE 流式传输

### 4.3 用户系统（lib/auth.ts + lib/db.ts）

简约架构：
- JWT 令牌认证（30 天有效期）
- JSON 文件数据库（data/db.json）
- 免费用户每日限额控制

### 4.4 会员支付（app/api/payment/）

两级会员：
- Pro 会员（29.9 元/月）：无限排盘 + AI 解读 + 合盘
- 终身会员（99.9 元）：永久权益
